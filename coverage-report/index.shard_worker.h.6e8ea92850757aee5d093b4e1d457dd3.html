4 // v5: 多核并行 — 共享 Dict + SpinLock
5 // ============================================================
6 // 设计思路:
7 // 1. N 个 worker 线程，每个独立 epoll 事件循环
8 // 2. SO_REUSEPORT 内核级连接分发
9 // 3. 所有线程共享一个 Dict，用 SpinLock 保护
10 // 4. 临界区只有 Dict 操作（~200ns），自旋等待远快于 mutex 阻塞
11 //
12 // 为什么 SpinLock 而不是 Mutex:
13 // - Mutex: 阻塞线程 → 内核调度 → ~5-10μs 唤醒延迟
14 // - SpinLock: 用户态自旋 → ~200ns 等到锁 → 零上下文切换
15 // - 临界区越短，SpinLock 优势越大
16 //
17 // 为什么不用 fiber (v1-v4):
18 // - fiber yield 是协作式，持有锁时 yield → 死锁
19 // - 线程是抢占式，持有锁时被抢占 → OS 会调度其他线程 → 最终释放
20 //
21 // 与 v1-v4 的区别:
22 // - v1-v4: 单线程 fiber，无锁，30万 rps
23 // - v5: 多线程 epoll + SpinLock，预期 4 核 → 80-100万 rps
24 //
